# Slack共有用テキスト（ミニ ChatGPT クローン）

そのままコピペで使えます。

---

🤖 ミニ ChatGPT クローンを作りました（もくもく会の成果）

前回のTwitterクローンで「アプリの作り方の基本」を学んだ続きとして、今回は「アプリにAIを組み込む」ための教材（ChatGPTクローン）を、1〜14章ぜんぶ作りました。
TypeScript + Express + OpenAI + Supabase で、本物と同じ構造のチャットを実装しています。

▼ コード（GitHub）
https://github.com/shoujiki-panman/mini-chatgpt-clone

▼ できること
サーバー経由でLLMと会話 / 会話が続く（履歴を毎回送り直す） / Googleログイン＋会話をDB保存(RLS) /
使いすぎ防止(サーバー側で1日上限・超過は429) / トークンあふれ対策(要約＋直近だけ送る) /
メモリ(会話をまたいで名前・好みを覚える) / ストリーミング(1文字ずつ) / ツール(電卓で正確な計算)

▼ いちばんの学び（背骨は2つ）
①「LLMは毎回忘れる」= 会話が続くのは、履歴（要約＋直近）を毎回送り直しているから。記憶を作るのは開発者。
②「サーバーで鍵と財布を守る」= APIキーはブラウザに出さない。使いすぎはサーバー側で止める。
そして「APIをつなぐだけ」では足りない、と体で理解しました（計算もできないし、実在URLも出せない → ツールが要る）。
金額・使わせない工夫・記憶のさせ方まで学んでから実装するのが大事だと実感。前回が「アプリの基本」なら、今回は「AIを組み込む第二章」として本当に良い教材でした。

▼ つまづきどころ（共有）
APIキーはターミナルでなくエディタで.envに書く / OpenAIは前払い式(少額チャージが要る) /
Supabase無料は2プロジェクトまで(Twitterクローンに相乗り) / Googleログインの戻り先URL追加 /
動くコードは<script>の中に置く / ブラウザ用(CDN)とサーバー用(npm)は別 / SSEで改行が消える→断片をJSON化

▼ これから始める人へ
中島聡さん/Singularity Societyの「ChatGPTクローン教材」は、AIをアプリに組み込みたい人の最初の一歩に本当に良いです。
背骨（記憶・鍵・使いすぎ）は、Gemini Liveなど他のAIを使うときもそのまま効きます。
